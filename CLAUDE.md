# VoxShare — Agent Instructions

## What This App Does

Session-based voice tracker: measures how much each participant speaks (time, turns, talk ratio). Fully on-device — no audio leaves the browser, no API keys required. Deployable as a PWA to any static host.

## Critical Implementation Rules

### vite.config.ts
- Always set COOP/COEP headers on the dev server (`sw.js` only kicks in after the first SW registration):
  ```ts
  server: {
    headers: {
      'Cross-Origin-Opener-Policy': 'same-origin',
      'Cross-Origin-Embedder-Policy': 'require-corp',
    }
  }
  ```
- A custom `staticWorkerAssetsPlugin` serves `ort-wasm-simd-threaded.mjs` and `vad.worklet.bundle.min.js` as plain static assets in dev, bypassing Vite's transform pipeline (required for AudioWorklet/WASM worker scripts).
- VAD/WASM assets are copied from `node_modules` to `public/` by `scripts/copy-vad-assets.mjs` — do not use `vite-plugin-static-copy`.
- `VitePWA` is configured with `selfDestroying: true` — it generates only the PWA manifest; all caching is handled by `public/sw.js`.

### UI components (`src/components/ui/`)
- Always use shadcn/ui primitives for generic UI elements (buttons, inputs, cards, badges, tables, scroll areas, etc.)
- Add new components via `pnpm dlx shadcn@latest add <component>` — this drops the file into `src/components/ui/` with the correct Radix + CVA wiring
- Do not hand-roll primitives that shadcn already covers

### index.html
- The inline `<script>` registers `sw.js` and handles cross-origin isolation reloads.
- It calculates the base path correctly for GitHub Pages deployment.

### lib/mfcc.ts
- Frame size 2048, hop 1024, 13 MFCC coefficients
- Always set `Meyda.bufferSize`, `Meyda.sampleRate`, and `Meyda.numberOfMFCCCoefficients` before each `extract()` call (Meyda uses global state)
- Average vectors across all frames → one profile vector per speaker
- `identifySpeaker`: return speaker id if best cosine similarity ≥ 0.75, else `null`

### lib/vad.ts
```ts
MicVAD.new({
  baseAssetPath: '/',
  onnxWASMBasePath: '/',
  model: 'legacy',
  minSpeechMs: 300,
  preSpeechPadMs: 200,
  positiveSpeechThreshold: 0.6,
  negativeSpeechThreshold: 0.45,
})
```

### lib/enrollment.ts
- Records 5 seconds of audio via `MediaRecorder`
- Use `AudioContext({ sampleRate: 16000 }).decodeAudioData()` to force-resample to 16 kHz
- iOS Safari: fall back to `audio/mp4` mimeType if `audio/webm` is unsupported

### store/sessionStore.ts — key actions
| Action | When |
|---|---|
| `recordSpeechStart(speakerId)` | VAD `onSpeechStart` — opens a pending turn |
| `recordTurn(speakerId, durationMs)` | VAD `onSpeechEnd` — closes turn when speaker identified |
| `recordUnknownTurn(durationMs)` | VAD `onSpeechEnd` — closes turn when speaker not identified |
| `endSession()` | "End Session" button — auto-closes any open turn |
| `reset()` | Restart flow from scratch |

## Gotchas

- **WASM filenames change** across `onnxruntime-web` versions — after install, check `node_modules/onnxruntime-web/dist/` and update `scripts/copy-vad-assets.mjs` to match
- **Meyda global state** — set `bufferSize`, `sampleRate`, and `numberOfMFCCCoefficients` before every `extract()` call
- **Biome does not type-check** — always run `pnpm typecheck` separately; Biome only covers syntax/style
- **COEP blocks cross-origin resources** — no cross-origin `<img>`, `<script>`, or fetch without proper `crossorigin` attribute and `Cross-Origin-Resource-Policy` response header
- **sw.js causes one reload on first visit** — expected behavior; SW registers then page reloads with isolation active
- **Speaker threshold tuning** — noisy/echoic rooms may need threshold lowered from 0.75 to 0.65–0.70
- **iOS PWA: background audio stops when screen locks** — acceptable; device stays visible during meetings

## Verification Checklist

Before marking any phase complete, verify:

- `window.crossOriginIsolated === true` in DevTools console
- Network tab: `silero_vad_legacy.onnx` and `*.wasm` → 200
- `pnpm check` exits 0
- `pnpm typecheck` exits 0
- VAD fires: speak → `onSpeechEnd` logs a Float32Array in console
- Enrollment: `store.participants[i].mfccProfile` is a 13-element non-zero array
- Same-speaker cosine similarity ≥ 0.80; different-speaker ≤ 0.65
